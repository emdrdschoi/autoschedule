# SHIFT SCHEDULER 사용 설명서

의료진 D/E/N 근무표를 자동 생성하는 스케줄러입니다.  
날짜별 필요 인원, 개인별 근무 불가/희망, 연차, 야간 후 휴식, 연속근무 제한, grade 기반 고년차 배정, 저년차 분산, 개인별 D/E/N/Total 고정 개수, 결과표 셀 고정 등을 반영하여 OR-Tools CP-SAT solver로 근무표를 만듭니다.

---

## 1. 핵심 개념

| 항목 | 의미 | 예시 |
|---|---|---|
| Duty 설정 | 날짜별 D/E/N 필요 인원 | 8/1 D 3명, E 2명, N 1명 |
| 근무 요청 / 날짜 설정 | 개인별 특정 날짜의 불가/희망/연차 입력 | `d`, `x`, `a`, `N` |
| 개인 규칙 / Grade | 개인별 grade, 근무 조정값, D/E/N/Total 고정, 근무 패턴 제한 | grade 8, Total 고정 10, N 후 2일 off |
| Grade 정책 | 전체 스케줄에 적용되는 고년차/저년차 기준 | grade ≥ 6이면 고년차, duty마다 고년차 1명 이상 |
| 결과표 셀 고정 | 결과표에서 선택한 셀을 다음 실행에도 유지 | 특정 날짜 특정 의사를 D로 고정 |
| Soft penalty | 가능한 한 공평하게 맞추는 기준 | 총근무, N, 휴일, D/E, grade 편차 |

### Hard rule과 Soft rule

| 종류 | 설명 | 예시 |
|---|---|---|
| Hard rule | 반드시 지켜야 하는 조건. 불가능하면 해가 나오지 않음 | Duty 필요 인원, 대문자 D/E/N 강제, x/a 불가, fixed_Total, 고년차 최소 인원 |
| Soft rule | 가능하면 맞추는 조건. 어기면 penalty가 붙음 | 총근무 균형, N 균형, 휴일 균형, 저년차 초과 배치 최소화 |

## 2. 전체 작업 순서

```text
1. 사이드바에서 시작 날짜와 총 일수 설정
2. 의사 이름을 직접 추가하거나 설정 Excel 불러오기
3. 근무 요청 / 날짜 설정 탭에서 날짜 유형과 개인 요청 입력
4. Duty 설정 탭에서 날짜별 D/E/N 필요 인원 입력
5. 개인 규칙 / Grade 탭에서 grade, 근무 조정값, fixed_Total, 개인 규칙 입력
6. Duty 설정 탭 상단의 Duty 총합 / fixed_total 합 / 남은 근무수 확인
7. 스케줄 생성
8. 결과 탭에서 근무 통계, 근무 일정표, Daily duty 구성 요약 확인
9. 필요한 셀을 고정한 뒤 재실행하거나 Excel로 내보내기
```

처음 사용할 때는 의사 이름만 넣고 `설정 Excel 저장`을 먼저 눌러 템플릿을 만든 뒤, Excel에서 대량 편집 후 다시 불러오는 방식이 가장 편합니다.

> 주의: 설정 Excel의 날짜 자체가 앱의 시작 날짜를 자동 변경하지는 않습니다. 달력 기준은 사이드바의 `시작 날짜`와 `총 일수`입니다.

## 3. 사이드바

### 3.1 기본 설정

| 항목 | 설명 |
|---|---|
| 시작 날짜 | 스케줄의 첫 날짜 |
| 총 일수 | 생성할 날짜 수. 보통 한 달 단위 |

시작 날짜를 바꾸면 토/일 날짜 유형이 자동 재계산됩니다.

### 3.2 의사 관리

- 의사 이름을 추가, 수정, 삭제할 수 있습니다.
- 실제 운영 중에는 삭제보다 Excel에서 정리 후 다시 불러오는 편이 안전합니다.
- Grade, 근무 조정값, fixed_Total 등은 `개인 규칙 / Grade` 탭에서 설정합니다.

### 3.3 솔버 설정

| 모드 | 설명 |
|---|---|
| 최적해 1개 | soft penalty가 가장 낮은 해 하나를 찾습니다. 가장 빠르고 안정적입니다. |
| 다중 솔루션 탐색 | 먼저 최적 penalty를 찾고, 허용 범위 안의 여러 후보를 찾아 비교합니다. |

다중 솔루션은 2단계로 작동합니다.

```text
1단계: solver가 먼저 soft rule penalty의 최적값(best_adv)을 찾음
2단계: adv <= best_adv + 추가 허용 편차 범위 안에서 여러 solution 탐색
```

예:

```text
최적 편차(best_adv) = 21
추가 허용 편차 = 0  → adv <= 21인 최적해만 탐색
추가 허용 편차 = 2  → adv <= 23까지 허용하여 조금 더 다양한 후보 탐색
```

## 4. 근무 요청 / 날짜 설정 탭

이 탭에서는 날짜 유형과 개인별 근무 요청을 입력합니다.

### 4.1 날짜 유형

| 값 | 의미 | 휴일 통계 포함 |
|---|---|---|
| 평일 | 일반 평일 | 아니오 |
| 토 | 토요일 | 예 |
| 일 | 일요일 | 예 |
| 공 | 평일 공휴일 | 예 |

결과표에서는 토/일/공휴일 열이 음영 처리됩니다. 평일 공휴일은 날짜 옆에 `*` 또는 휴일 표시가 붙습니다.

### 4.2 개인 근무 요청 입력값

| 입력 | 의미 | solver 처리 |
|---|---|---|
| 빈칸 | 특별 요청 없음 | 자동 배정 가능 |
| `d` | Day 불가 | D 불가 |
| `e` | Evening 불가 | E 불가 |
| `n` | Night 불가 | N 불가 |
| `de` | Day, Evening 불가 | D/E 불가 |
| `dn` | Day, Night 불가 | D/N 불가 |
| `en` | Evening, Night 불가 | E/N 불가 |
| `den` | D/E/N 모두 불가 | 전체 근무 불가 |
| `x` | 해당 날짜 전체 불가 | 전체 근무 불가 |
| `a` | 연차 | 전체 근무 불가 + 연차 통계 표시 + 평준화에서 -1 효과 |
| `D` | Day 희망/강제 | 해당 날짜 D로 강제 |
| `E` | Evening 희망/강제 | 해당 날짜 E로 강제 |
| `N` | Night 희망/강제 | 해당 날짜 N으로 강제 |

대문자 `D/E/N`은 hard rule입니다. 너무 많이 넣으면 해가 나오지 않을 수 있습니다.

### 4.3 연차 `a`의 의미

`a`는 off입니다. 즉 solver 내부에서는 `x`처럼 D/E/N 모두 불가로 처리합니다.  
다만 통계와 평준화에서는 연차로 따로 인식합니다.

```text
a 1개 = 그 사람은 해당 날짜 근무 불가
       + 근무 통계의 연차 컬럼에 1개 표시
       + fixed_Total이 없는 경우 근무 조정값 -1과 비슷하게 작동
```

예:

```text
A 의사: 연차 a 2개, fixed_Total 없음
→ 다른 사람보다 목표 근무수가 약 2개 적게 잡히도록 평준화
```

중요:

```text
fixed_Total이 지정된 사람에게는 연차 a가 fixed_Total을 줄이지 않습니다.
```

예:

```text
fixed_Total = 10
연차 a = 2개
→ 실제 D/E/N 총근무는 여전히 10개로 맞춤
→ 연차는 통계에 2개로 표시
```

## 5. Duty 설정 탭

날짜별로 필요한 D/E/N 인원을 입력합니다.

예:

| 날짜 | D | E | N |
|---|---:|---:|---:|
| 8/1 | 3 | 2 | 1 |
| 8/2 | 3 | 2 | 1 |
| 8/3 | 2 | 1 | 1 |

필요 인원이 0인 duty에는 아무도 배정되지 않습니다.

### 5.1 Duty 총합 / fixed_total 요약

Duty 설정 탭 상단에는 항상 다음 정보가 표시됩니다.

```text
Duty 총합
D/E/N 각각의 총 필요 근무수
fixed_total 합
fixed_total 미지정 인원
차이
남은 근무수
```

`남은 근무수 = Duty 총합 - fixed_total 합`입니다.

| 상황 | 의미 | 해야 할 일 |
|---|---|---|
| 차이 `0` | Duty 총합과 fixed_total 합이 정확히 맞음 | 그대로 진행 가능 |
| 차이 `+2` | Duty 총합이 fixed_total 합보다 2개 많음 | fixed_Total 없는 인원이 있으면 자동 배분. 모든 인원이 fixed_Total이면 Duty를 2개 줄이거나 fixed_Total을 2개 늘려야 함 |
| 차이 `-2` | fixed_total 합이 Duty 총합보다 2개 많음 | Duty를 2개 늘리거나 fixed_Total을 2개 줄여야 함 |

이 요약은 solver를 돌리기 전에 먼저 확인하는 것이 좋습니다.

## 6. 개인 규칙 / Grade 탭

### 6.1 Grade 정책 설정

Grade는 개인의 숙련도/연차를 숫자로 표현한 값입니다. 현재 입력 범위는 1~10입니다.

| 항목 | 의미 | 예시 |
|---|---|---|
| 고년차 기준 grade ≥ | 이 grade 이상이면 고년차 | 6 |
| Duty별 고년차 최소 | 각 D/E/N duty마다 필요한 고년차 최소 인원 | 1 |
| 저년차 기준 grade ≤ | 이 grade 이하이면 저년차 | 3 |
| Duty별 저년차 권장 최대 | 한 duty에 들어가도 좋은 저년차 수 | 1 |
| 저년차 초과 penalty | 저년차가 권장 최대를 넘을 때 penalty | 1 |

고년차 최소 인원은 hard rule입니다. 예를 들어 `고년차 기준 grade ≥ 6`, `Duty별 고년차 최소 1`이면 모든 D/E/N duty에 grade 6 이상이 최소 1명 필요합니다.

저년차 제한은 soft rule입니다. 초과하면 penalty가 붙지만, 다른 조건 때문에 필요하면 허용될 수 있습니다.

### 6.2 편차 가중치 설정

| 항목 | 의미 |
|---|---|
| D/E 편차 가중치 | D와 E 개수 불균형을 줄이는 정도 |
| 휴일 편차 가중치 | 토/일/공휴일 근무 불균형을 줄이는 정도 |
| 총 근무 편차 가중치 | 전체 근무 개수 불균형을 줄이는 정도 |
| N 편차 가중치 | Night 개수 불균형을 줄이는 정도 |
| Grade 편차 가중치 | duty별 평균 grade가 너무 낮거나 높게 치우치지 않게 하는 정도 |

가중치를 높이면 해당 균형을 더 강하게 맞추려고 합니다.

### 6.3 개인별 Grade

각 의사의 grade를 입력합니다.

| Grade | 의미 예시 |
|---:|---|
| 1~3 | 저년차 |
| 4~5 | 중간 |
| 6~8 | 고년차 |
| 9~10 | 최고년차/책임 가능 인력 |

실제 의미는 병원 상황에 맞게 정하면 됩니다. 중요한 것은 Grade 정책에서 고년차/저년차 기준을 어떻게 잡는지입니다.

### 6.4 근무 조정값

근무 조정값은 평균보다 몇 개 더 또는 덜 일하게 할지 조정하는 값입니다.

| 값 | 의미 |
|---:|---|
| 0 | 평균대로 |
| +1 | 평균보다 약 1개 더 |
| -1 | 평균보다 약 1개 덜 |

연차 `a`는 fixed_Total이 없는 사람에 대해 이 근무 조정값을 자동으로 더 줄이는 효과를 냅니다.

예:

```text
근무 조정값 = 0, 연차 a 2개
→ 평준화 계산에서는 대략 -2처럼 반영
```

### 6.5 Total/D/E/N 고정 개수

특정 의사의 전체 근무 수 또는 D/E/N 개수를 정확히 고정하고 싶을 때 사용합니다.

| 값 | 의미 |
|---:|---|
| -1 | 자동 평준화 |
| 0 | 해당 항목을 0개로 고정 |
| 3 | 해당 항목을 정확히 3개로 고정 |

예:

| Total 고정 | D 고정 | E 고정 | N 고정 | 의미 |
|---:|---:|---:|---:|---|
| -1 | -1 | -1 | -1 | 모두 자동 평준화 |
| 8 | -1 | -1 | -1 | 총근무는 정확히 8개, D/E/N 구성은 자동 |
| -1 | 3 | -1 | -1 | D는 정확히 3개, E/N/Total은 자동 |
| 10 | 3 | 3 | 4 | 총 10개, D/E/N도 3/3/4개로 고정 |

주의:

- fixed_Total은 hard rule입니다.
- fixed_D/E/N 합이 fixed_Total보다 크면 불가능합니다.
- fixed_D/E/N을 모두 고정한 경우 합이 fixed_Total과 같아야 합니다.
- 연차 `a`는 fixed_Total 값을 줄이지 않습니다.

### 6.6 개인별 근무 규칙

| 규칙 | 의미 |
|---|---|
| 하루 근무 횟수 | 하루에 D/E/N을 몇 개까지 허용할지 |
| N 뭉치 최대 길이 | 연속 Night 허용 길이 |
| N뭉치 후 완전 Off 의무일 | Night block 후 반드시 쉬어야 하는 날짜 수 |
| N뭉치 후 다음 N까지 총 간격 | 다음 Night까지 최소 간격 |
| Evening 후 Day 금지 | E 다음날 D 금지 |
| Evening 3연속 금지 | EEE 금지 |
| 4일내 Evening 3회 금지 | 4일 중 E 3회 이상 방지 |
| 최대 연속 근무일수 | 입력한 일수까지 연속 근무 허용 |
| 7일 구간 최대 근무수 | 어느 7일 구간에서도 허용할 최대 근무수 |
| Day 3연속 금지 | DDD 금지 |

`최대 연속 근무일수 = 4`이면 4일 연속 근무까지 허용되고, 5일 연속 근무는 금지됩니다.

## 7. 결과 탭

### 7.1 근무 통계

근무 통계는 Name 정렬이 아니라 입력 순서 그대로 표시됩니다.  
맨 왼쪽 `No`는 `근무 요청 / 날짜 설정` 탭 및 Excel 입력 순서 기준입니다.

현재 주요 컬럼은 다음과 같습니다.

| 컬럼 | 의미 |
|---|---|
| No | 입력 순서 index |
| Name | 의사 이름 |
| Grade | 개인 grade |
| Senior | 고년차 여부 또는 관련 표시 |
| Junior | 저년차 여부 또는 관련 표시 |
| D | Day 근무 수 |
| E | Evening 근무 수 |
| N | Night 근무 수 |
| Total | D+E+N 총근무 수 |
| 연차 | 근무 요청의 `a` 개수 |
| Holiday | 토/일/공휴일 근무 수 |
| Fri_N | 금요일 Night 수 |
| 주간평균hr | 주당 평균 근무시간 추정치 |

`Tue_N`은 제거되었습니다.

### 7.2 편차 / penalty 정보

결과 화면에는 objective 구성요소가 표시됩니다.

```text
편차*가중치 합
최적 편차
허용 상한
balance penalty
k D/E
k1 휴일
k2 총근무
k3 N
k4 Grade편차
저년차 초과 penalty
```

값이 낮을수록 solver가 더 선호한 근무표입니다. 다만 실제 운영에서는 숫자만 보지 말고 특정 날짜의 인력 구성도 함께 확인해야 합니다.

### 7.3 근무 일정표 / 셀 선택 & 고정

근무 일정표는 Grade로 정렬하지 않고 입력 순서 그대로 표시됩니다.  
다만 이름 옆에 grade는 함께 표시됩니다.

```text
Kim [G8]
Lee [G3]
Park [G6]
```

날짜 열은 공간을 줄이기 위해 짧게 표시됩니다.

```text
1토, 2일, 3월, 4화, 5수, 6목, 7금
```

평일 공휴일은 `*`가 붙어 구분됩니다.

| 표시 | 의미 |
|---|---|
| D | Day |
| E | Evening |
| N | Night |
| A | 연차 |
| · | Off |
| 🔒D / 🔒E / 🔒N / 🔒· / 🔒A | 결과표에서 고정된 셀 |

### 7.4 셀 고정 / 고정 해제

결과표에서 셀을 선택하고 고정하면 다음 실행에서도 유지됩니다.

| 기능 | 설명 |
|---|---|
| 🔒 선택 셀 고정 | 현재 결과값을 fixed layer로 저장합니다. D/E/N은 해당 근무로, off는 `x`, 연차는 `a`로 고정됩니다. |
| 🔓 선택 셀 고정 해제 | fixed layer만 제거합니다. 기존 사용자가 입력했던 `d`, `en`, 빈칸 등은 다시 살아납니다. |

내부적으로 근무 요청은 두 층으로 관리됩니다.

```text
base_shift_requests  = 사용자가 근무 요청 탭 또는 Excel에서 직접 입력한 원래 값
fixed_shift_requests = 결과표에서 셀을 선택해 추가로 고정한 값
solver 입력값         = base_shift_requests 위에 fixed_shift_requests를 덮어쓴 값
```

예:

```text
원래 입력: en
결과표에서 D로 고정
→ solver에는 D가 들어감
고정 해제
→ 원래 en이 다시 적용됨
```

### 7.5 Daily duty 구성 요약

날짜별 D/E/N duty마다 실제 배정 인원, 필요 인원, 평균 grade, 근무자를 보여줍니다.

근무자 이름 옆에는 grade가 같이 표시됩니다.

```text
Kim [8], Lee [3], Park [6]
```

주요 컬럼:

| 컬럼 | 의미 |
|---|---|
| 날짜 | 날짜와 요일 |
| 유형 | 평일/토/일/공 |
| Duty | D/E/N |
| 인원/필요 | 실제 배정 인원 / 필요 인원 |
| 평균 Grade | 해당 duty 근무자의 평균 grade |
| 근무자 | 이름 [grade] 목록 |

토/일/공휴일 row는 음영 처리됩니다.

### 7.6 다중 솔루션 이동

`이전`, `다음`, `이동` 기능으로 이미 찾은 여러 후보를 비교할 수 있습니다.  
이미 계산된 solution을 넘겨보는 것이므로 solver를 새로 실행하지 않습니다.

### 7.7 결과 Excel 내보내기

결과 Excel은 화면 전환 때마다 자동 생성하지 않고, `Excel 준비` 버튼을 눌렀을 때 생성됩니다.  
준비가 끝나면 다운로드 버튼으로 저장합니다.

## 8. Excel 설정 파일 구조

`설정 Excel 저장`을 누르면 현재 설정을 Excel로 저장할 수 있습니다.  
수정 후 다시 불러오면 같은 조건으로 스케줄을 생성할 수 있습니다.

### 8.1 Doctors sheet

| 컬럼 | 의미 |
|---|---|
| name | 의사 이름 |
| shift_adj | 근무 조정값 |
| grade | 개인 grade, 1~10 권장 |

### 8.2 Rules sheet

첫 번째 열은 의사 이름입니다. 각 행은 한 의사의 개인 rule입니다.

주요 컬럼:

```text
rule_max_shifts_per_day
rule_n_block_max
rule_n_rest
rule_n_gap
rule_no_day_after_eve
rule_no_3eve_consec
rule_no_3eve_in_4days
rule_max_consec_days
rule_max_shifts_per_week
rule_no_3day_consec
fixed_D
fixed_E
fixed_N
fixed_Total
```

`fixed_D/E/N/Total`은 `-1`이면 자동 평준화입니다.

### 8.3 GradeRules sheet

전체 grade 정책과 objective 가중치입니다.

| key | 의미 |
|---|---|
| senior_min_grade | 고년차 기준 grade 이상 |
| senior_min_count | 각 duty별 고년차 최소 인원 |
| junior_max_grade | 저년차 기준 grade 이하 |
| junior_soft_max_count | 각 duty별 저년차 권장 최대 인원 |
| junior_penalty_weight | 저년차 초과 penalty 가중치 |
| weight_de_dev | D/E 편차 가중치 |
| weight_holiday_dev | 휴일 편차 가중치 |
| weight_total_dev | 총근무 편차 가중치 |
| weight_n_dev | N 편차 가중치 |
| weight_grade_dev | Grade 편차 가중치 |

### 8.4 DutyRequests sheet

행은 날짜, 열은 D/E/N입니다.

| 날짜 | D | E | N |
|---|---:|---:|---:|
| 2026-08-01 | 3 | 2 | 1 |

### 8.5 ShiftRequests sheet

행은 의사, 열은 날짜입니다. 첫 번째 열은 의사 이름입니다.

| Name | 2026-08-01 | 2026-08-02 | 2026-08-03 |
|---|---|---|---|
| Kim |  | d | a |
| Lee | x |  | N |

입력 가능한 값은 앱의 근무 요청 탭과 동일합니다.

```text
빈칸, d, e, n, de, dn, en, den, x, a, D, E, N
```

### 8.6 FixedShiftRequests sheet

결과표에서 셀 고정한 값이 저장되는 시트입니다.  
사용자가 직접 입력한 원래 요청은 `ShiftRequests`, 결과표 고정값은 `FixedShiftRequests`에 분리 저장됩니다.

## 10. 추천 운영 방식

### 10.1 처음 세팅할 때

```text
1. 의사 이름을 모두 추가
2. 설정 Excel 저장
3. Excel에서 Doctors, Rules, DutyRequests, ShiftRequests 편집
4. 다시 불러오기
5. 앱에서 Grade 정책, fixed_Total, Duty 총합 요약 확인
6. 스케줄 생성
```

### 10.2 연차가 있는 달

```text
1. ShiftRequests에 연차 날짜를 a로 입력
2. fixed_Total이 없는 사람은 연차만큼 목표 근무수가 줄어듦
3. fixed_Total이 있는 사람은 Total이 줄지 않으므로 필요하면 fixed_Total을 직접 조정
4. 결과 통계에서 연차 컬럼 확인
```

### 10.3 fixed_Total을 많이 쓰는 달

```text
1. 개인별 Total 고정 입력
2. Duty 설정 탭 상단의 차이 확인
3. 차이 0이면 가장 안전
4. 차이 +값이면 남은 근무가 fixed_Total 없는 사람에게 배분됨
5. 차이 -값이면 불가능하므로 Duty를 늘리거나 fixed_Total을 줄임
```

## 11. 해가 안 나올 때 확인할 것

1. Duty 총합보다 fixed_total 합이 크지 않은지 확인합니다.
2. 모든 사람에게 fixed_Total을 넣었다면 합이 Duty 총합과 정확히 같은지 확인합니다.
3. fixed_D/E/N 합이 fixed_Total보다 크지 않은지 확인합니다.
4. 고년차 최소 인원이 duty 필요 인원보다 크지 않은지 확인합니다.
5. 고년차 기준에 해당하는 사람이 충분한지 확인합니다.
6. 대문자 `D/E/N` 강제 요청이 너무 많지 않은지 확인합니다.
7. `x`, `a`, `den`으로 전체 불가인 날이 너무 많지 않은지 확인합니다.
8. N 후 휴식, 최대 연속 근무, 7일 구간 최대 근무수가 너무 엄격하지 않은지 확인합니다.
9. solver 탐색 시간을 늘려봅니다.

## 12. 결과가 마음에 안 들 때 조정법

| 문제 | 조정할 항목 |
|---|---|
| 총근무 수가 불공평 | 총 근무 편차 가중치 증가 |
| Night가 불공평 | N 편차 가중치 증가 |
| 휴일이 불공평 | 휴일 편차 가중치 증가 |
| D/E가 불공평 | D/E 편차 가중치 증가 |
| duty별 grade 평균이 너무 낮음 | Grade 편차 가중치 증가, 고년차 최소 인원 조정 |
| 저년차가 너무 몰림 | 저년차 penalty 증가 |
| 저년차 penalty가 너무 지배적 | 저년차 penalty 감소 |
| 특정 사람이 평균보다 많이/적게 해야 함 | 근무 조정값 또는 fixed_Total 조정 |
| 연차자가 너무 많이 배정되는 느낌 | 연차는 `a`, 전체 불가는 `x`로 정확히 입력했는지 확인 |

## 13. 주의사항

- 이 프로그램은 최적화 도구이며 최종 승인자가 아닙니다. 생성된 근무표는 반드시 실제 운영자가 검토해야 합니다.
- Hard rule이 너무 많으면 해가 없을 수 있습니다.
- Soft rule은 완벽히 지켜지는 조건이 아니라 penalty를 통해 유도되는 조건입니다.
- 다중 솔루션의 1단계 최적 편차는 solver가 주어진 탐색 시간 안에서 찾은 최선값입니다. 탐색 시간이 너무 짧으면 더 좋은 해가 있을 수 있습니다.
- Excel에서 의사 이름을 바꾸면 ShiftRequests / FixedShiftRequests 매칭이 달라질 수 있습니다.
- 앱 화면에서 설정 Excel을 불러온 뒤에는 Duty 설정 탭과 개인 규칙 / Grade 탭에 값이 제대로 반영되었는지 확인하세요.
- grade는 1~10 범위로 사용하는 것을 권장합니다.
