# SHIFT SCHEDULER 사용 설명서

의료진 D/E/N 근무표를 자동 생성하는 스케줄러입니다.  
날짜별 필요 인원, 개인별 근무 불가/희망, 연차, 야간 후 휴식, 연속근무 제한, grade 기반 고년차 배정, 저년차 분산, 초저년차 동시근무 hard rule, 개인별 D/E/N/Total 고정 개수, 결과표 셀 고정 등을 반영하여 OR-Tools CP-SAT solver로 근무표를 만듭니다.

---

## 1. 핵심 개념

| 항목 | 의미 | 예시 |
|---|---|---|
| Duty 설정 | 날짜별 D/E/N 필요 인원 | 8/1 D 3명, E 2명, N 1명 |
| 근무 요청 / 날짜 설정 | 개인별 특정 날짜의 불가/희망/연차 입력 | `d`, `x`, `a`, `N` |
| 개인 규칙 / Grade | 개인별 grade, 근무 조정값, D/E/N/Total 고정, 근무 패턴 제한 | grade 8, Total 고정 10, N 후 2일 off |
| Grade 정책 | 전체 스케줄에 적용되는 고년차/저년차/초저년차 기준 | grade ≥ 6이면 고년차, grade ≤ 1 초저년차 duty당 최대 허용 인원 설정 |
| 결과표 셀 고정 | 결과표에서 선택한 셀을 다음 실행에도 유지 | 특정 날짜 특정 의사를 D로 고정 |
| Soft penalty | 가능한 한 공평하게 맞추는 기준 | 총근무, N, 휴일, D/E, grade 편차 |

### Hard rule과 Soft rule

| 종류 | 설명 | 예시 |
|---|---|---|
| Hard rule | 반드시 지켜야 하는 조건. 불가능하면 해가 나오지 않음 | Duty 필요 인원, 대문자 D/E/N 강제, x/a 불가, fixed_Total, 고년차 최소 인원, 초저년차 최대 허용 인원 제한 |
| Soft rule | 가능하면 맞추는 조건. 어기면 penalty가 붙음 | 총근무 균형, N 균형, 휴일 균형, 저년차 초과 배치 최소화 |

## 2. 전체 작업 순서

```text
1. 사이드바에서 시작 날짜와 총 일수 설정
2. 의사 이름을 직접 추가하거나 설정 Excel 불러오기
3. 근무 요청 / 날짜 설정 탭에서 날짜 유형과 개인 요청 입력
4. Duty 설정 탭에서 날짜별 D/E/N 필요 인원 입력
5. 개인 규칙 / Grade 탭에서 grade, 근무 조정값, fixed_D/E/N/Total, 개인 규칙 입력
6. Duty 설정 탭 상단의 Duty 총합 / fixed_total 합 / 남은 근무수 확인
7. 스케줄 생성
8. 결과 탭에서 근무 통계, 근무 일정표, Daily duty 구성 요약 확인
9. 필요한 셀을 고정한 뒤 재실행하거나 Excel로 내보내기
```

처음 사용할 때는 의사 이름만 넣고 `설정 Excel 저장`을 먼저 눌러 템플릿을 만든 뒤, Excel에서 대량 편집 후 다시 불러오는 방식이 가장 편합니다.

> 주의: 설정 Excel의 날짜 자체가 앱의 시작 날짜를 자동 변경하지는 않습니다. 달력 기준은 사이드바의 `시작 날짜`와 `총 일수`입니다.

## 3. 사이드바

### 3.1 기본 설정

| 항목 | 설명 |
|---|---|
| 시작 날짜 | 스케줄의 첫 날짜 |
| 총 일수 | 생성할 날짜 수. 보통 한 달 단위 |

시작 날짜를 바꾸면 토/일 날짜 유형이 자동 재계산됩니다.

### 3.2 의사 관리

- 의사 이름을 추가, 수정, 삭제할 수 있습니다.
- 실제 운영 중에는 삭제보다 Excel에서 정리 후 다시 불러오는 편이 안전합니다.
- Grade, 근무 조정값, fixed_D/E/N/Total 등은 `개인 규칙 / Grade` 탭에서 설정합니다.

### 3.3 솔버 설정

| 모드 | 설명 |
|---|---|
| 최적해 1개 | soft penalty가 가장 낮은 해 하나를 찾습니다. 가장 빠르고 안정적입니다. |
| 다중 솔루션 탐색 | 먼저 최적 penalty를 찾고, 허용 범위 안의 여러 후보를 찾아 비교합니다. |

다중 솔루션은 2단계로 작동합니다.

```text
1단계: solver가 먼저 soft rule penalty의 최적값(best_adv)을 찾음
2단계: adv <= best_adv + 추가 허용 편차 범위 안에서 여러 solution 탐색
```

예:

```text
최적 편차(best_adv) = 21
추가 허용 편차 = 0  → adv <= 21인 최적해만 탐색
추가 허용 편차 = 2  → adv <= 23까지 허용하여 조금 더 다양한 후보 탐색
```

## 4. 근무 요청 / 날짜 설정 탭

이 탭에서는 날짜 유형과 개인별 근무 요청을 입력합니다.

### 4.1 날짜 유형

| 값 | 의미 | 휴일 통계 포함 |
|---|---|---|
| 평일 | 일반 평일 | 아니오 |
| 토 | 토요일 | 예 |
| 일 | 일요일 | 예 |
| 공 | 평일 공휴일 | 예 |

결과표에서는 토/일/공휴일 열이 음영 처리됩니다. 평일 공휴일은 날짜 옆에 `*` 또는 휴일 표시가 붙습니다.

### 4.2 개인 근무 요청 입력값

| 입력 | 의미 | solver 처리 |
|---|---|---|
| 빈칸 | 특별 요청 없음 | 자동 배정 가능 |
| `d` | Day 불가 | D 불가 |
| `e` | Evening 불가 | E 불가 |
| `n` | Night 불가 | N 불가 |
| `de` | Day, Evening 불가 | D/E 불가 |
| `dn` | Day, Night 불가 | D/N 불가 |
| `en` | Evening, Night 불가 | E/N 불가 |
| `den` | D/E/N 모두 불가 | 전체 근무 불가 |
| `x` | 해당 날짜 전체 불가 | 전체 근무 불가 |
| `a` | 연차 | 전체 근무 불가 + 연차 통계 표시. 근무량 보정은 자동계산 버튼으로 화면에 계산한 뒤 저장으로 명시적 반영 가능 |
| `D` | Day 희망/강제 | 해당 날짜 D로 강제 |
| `E` | Evening 희망/강제 | 해당 날짜 E로 강제 |
| `N` | Night 희망/강제 | 해당 날짜 N으로 강제 |

대문자 `D/E/N`은 hard rule입니다. 너무 많이 넣으면 해가 나오지 않을 수 있습니다.

### 4.3 연차 `a`의 의미

`a`는 off입니다. 즉 solver 내부에서는 `x`처럼 D/E/N 모두 불가로 처리합니다.  
통계에서는 연차로 따로 표시되지만, 더 이상 solver 내부에서 자동으로 근무 조정값을 빼지는 않습니다.

```text
a 1개 = 그 사람은 해당 날짜 근무 불가
       + 근무 통계의 연차 컬럼에 1개 표시
       + 필요하면 자동계산 버튼으로 근무조정값을 화면에 계산한 뒤 저장으로 반영
```

예:

```text
A 의사: 연차 a 2개, fixed_Total 없음
→ 자동계산 버튼을 누르면 화면의 근무조정값 칸이 -2로 계산됨
→ 버튼을 누르지 않으면 a는 근무 불가 날짜와 연차 통계로만 작동
```

중요:

```text
fixed_Total이 지정된 사람에게는 연차 a가 fixed_Total을 줄이지 않습니다.
fixed_Total이 있는 사람의 자동 근무조정값은 fixed_Total - estimated average 기준으로 계산됩니다.
```

예:

```text
fixed_Total = 10
연차 a = 2개
→ 실제 D/E/N 총근무는 여전히 10개로 맞춤
→ 연차는 통계에 2개로 표시
→ 자동계산 버튼을 누르면 fixed_Total 기준 보정값이 화면의 근무조정값 칸에 계산됨
```

## 5. Duty 설정 탭

날짜별로 필요한 D/E/N 인원을 입력합니다.

예:

| 날짜 | D | E | N |
|---|---:|---:|---:|
| 8/1 | 3 | 2 | 1 |
| 8/2 | 3 | 2 | 1 |
| 8/3 | 2 | 1 | 1 |

필요 인원이 0인 duty에는 아무도 배정되지 않습니다.

### 5.1 Duty 총합 / fixed_total 요약

Duty 설정 탭 상단에는 항상 다음 정보가 표시됩니다. 같은 요약이 `개인 규칙 / Grade` 탭에도 표시되므로, fixed_Total을 조정하면서 바로 확인할 수 있습니다.

```text
Duty 총합
D/E/N 각각의 총 필요 근무수
fixed_total 합
fixed_total 미지정 인원
차이
남은 근무수
미지정 인원 1명당 자동 배분 예상 개수
```

`남은 근무수 = Duty 총합 - fixed_total 합`입니다. fixed_Total 미지정 인원이 있으면 `남은 근무수 ÷ 미지정 인원 수`를 계산해서 `(약 19.4개/명)`처럼 함께 보여줍니다. 이 값은 fixed_Total을 대략 몇 개로 넣으면 좋을지 판단하는 참고용입니다.

| 상황 | 의미 | 해야 할 일 |
|---|---|---|
| 차이 `0` | Duty 총합과 fixed_total 합이 정확히 맞음 | 그대로 진행 가능 |
| 차이 `+2` | Duty 총합이 fixed_total 합보다 2개 많음 | fixed_Total 없는 인원이 있으면 자동 배분되며, 1명당 예상 개수가 함께 표시됨. 모든 인원이 fixed_Total이면 Duty를 2개 줄이거나 fixed_Total을 2개 늘려야 함 |
| 차이 `-2` | fixed_total 합이 Duty 총합보다 2개 많음 | Duty를 2개 늘리거나 fixed_Total을 2개 줄여야 함 |

이 요약은 solver를 돌리기 전에 먼저 확인하는 것이 좋습니다. Duty 필요 인원이나 fixed_Total 입력값을 바꾸면 최신 number_input 값을 먼저 동기화한 뒤 계산하므로, 한 박자 늦은 숫자가 보이지 않도록 처리되어 있습니다.

## 6. 개인 규칙 / Grade 탭

### 6.1 fixed D/E/N/Total / Duty 총합 확인

> 복사/붙여넣기 편의를 위해 fixed count 편집 표는 결과 탭 근무 통계의 `D / E / N / Total` 순서와 동일하게 `fixed_D / fixed_E / fixed_N / fixed_Total` 순서로 표시됩니다.

개인 규칙 / Grade 탭에도 Duty 설정 탭과 같은 요약 박스가 표시됩니다.

```text
Duty 총합: 651 (D 217 / E 248 / N 186)
fixed_total 합: 117
fixed_total 미지정 인원: 29
차이: +534
남은 근무수: 534
💬 현재는 fixed_total 미지정 인원 29명에게 534개가 자동 배분됩니다 (약 18.4개/명).
```

해석은 다음과 같습니다.

```text
차이 +534
→ Duty 총합이 fixed_total 합보다 534개 많음
→ fixed_Total 미지정 인원 29명에게 약 18.4개/명으로 자동 배분됨
→ 모든 인원의 Total을 고정하려면 Duty를 534개 줄이거나 fixed_Total을 534개 늘려야 함

차이 -7
→ fixed_total 합이 Duty 총합보다 7개 많음
→ Duty를 7개 늘리거나 fixed_Total을 7개 줄여야 함
```

이 박스는 저장된 fixed count와 현재 Duty 필요 인원을 기준으로 계산됩니다. fixed count 표를 여러 칸 수정한 뒤에는 아래의 `fixed count 저장 / 요약에 반영` 버튼을 눌러야 요약, solve, 진단 입력값에 반영됩니다.

요약 박스에는 `연차(a) 총합`도 함께 표시되고, 바로 아래에는 개인별 연차(a), 근무 조정값과 `fixed_D / fixed_E / fixed_N / fixed_Total` 편집 표가 표시됩니다. 결과 탭의 근무 통계처럼 입력 순서 기준 `No / Name / Grade / Senior / Junior / 초저년차`를 함께 보여주고, `근무조정값`과 고정값 네 칸을 직접 수정할 수 있습니다.

```text
No | Name | Grade | Senior | Junior | 초저년차 | 연차(a) | 근무조정값 | fixed_D | fixed_E | fixed_N | fixed_Total
```

`fixed_D / fixed_E / fixed_N / fixed_Total` 값의 의미는 다음과 같습니다.

```text
빈칸 = 자동 평준화
0 = 해당 항목을 0개로 고정
1 이상 숫자 = 해당 의사의 D/E/N/Total 근무 수를 정확히 고정
```

이전 버전과의 호환을 위해 Excel에 `-1`이 들어 있어도 자동 평준화로 읽지만, 앱 화면과 새로 저장되는 Excel에서는 자동값을 빈칸으로 표시합니다.

이 표는 form 안에 있으므로 칸을 하나하나 바꿀 때마다 즉시 반영/갱신되지 않습니다. 여러 칸을 수정하거나 Excel에서 복사해 붙여넣은 뒤 `저장` 버튼을 누르면 그때 한 번에 저장되고, 위의 `Duty 총합 / 연차(a) 총합 / fixed_total 합 / 차이 / 남은 근무수` 요약과 solve/진단 입력값에 반영됩니다. `자동계산` 버튼을 누르면 현재 표 내용을 기준으로 권장 `근무조정값`을 화면 표에만 계산해 넣습니다. 이 단계에서는 아직 실제 설정에 저장되지 않으며, 확인 후 `저장` 버튼을 눌러야 위의 요약과 solve/진단 입력값에 반영됩니다. 기존처럼 아래 개인별 규칙 영역에 근무 조정값이나 D/E/N 고정 줄을 따로 두지 않고, 이 표에서 한 번에 관리합니다.

### 6.2 Grade 정책 설정

Grade는 개인의 숙련도/연차를 숫자로 표현한 값입니다. 현재 입력 범위는 1~10입니다.

| 항목 | 의미 | 예시 |
|---|---|---|
| 고년차 기준 grade ≥ | 이 grade 이상이면 고년차 | 6 |
| Duty별 고년차 최소 | 각 D/E/N duty마다 필요한 고년차 최소 인원 | 1 |
| 저년차 기준 grade ≤ | 이 grade 이하이면 저년차 | 3 |
| Duty별 저년차 권장 최대 | 한 duty에 들어가도 좋은 저년차 수 | 1 |
| 저년차 초과 penalty | 저년차가 권장 최대를 넘을 때 penalty | 1 |
| 초저년차 기준 grade ≤ | 이 grade 이하이면 초저년차 hard rule 대상 | 1 |
| 초저년차 최대 허용 | 초저년차가 같은 날짜·같은 D/E/N duty에 최대 몇 명까지 들어갈 수 있는지 설정. 0=사용안함, 1=최대 1명 허용 | 1 |

고년차 최소 인원은 hard rule입니다. 예를 들어 `고년차 기준 grade ≥ 6`, `Duty별 고년차 최소 1`이면 모든 D/E/N duty에 grade 6 이상이 최소 1명 필요합니다.

저년차 제한은 soft rule입니다. 초과하면 penalty가 붙지만, 다른 조건 때문에 필요하면 허용될 수 있습니다.

초저년차 최대 허용 인원 제한은 hard rule입니다. 예를 들어 `초저년차 기준 grade ≤ 1`, `초저년차 최대 허용 = 1`이면 grade 1 의사는 같은 날짜·같은 D/E/N duty에 최대 1명까지만 들어갈 수 있습니다. `초저년차 최대 허용 = 0`이면 이 hard rule은 적용하지 않습니다.

### 6.3 편차 가중치 설정

| 항목 | 의미 |
|---|---|
| D/E 편차 가중치 | D와 E 개수 불균형을 줄이는 정도 |
| 휴일 편차 가중치 | 토/일/공휴일 근무 불균형을 줄이는 정도 |
| 총 근무 편차 가중치 | 전체 근무 개수 불균형을 줄이는 정도 |
| N 편차 가중치 | Night 개수 불균형을 줄이는 정도 |
| Grade 편차 가중치 | duty별 평균 grade가 너무 낮거나 높게 치우치지 않게 하는 정도 |

가중치를 높이면 해당 균형을 더 강하게 맞추려고 합니다.

### 6.4 개인별 Grade

각 의사의 grade를 입력합니다.

| Grade | 의미 예시 |
|---:|---|
| 1~3 | 저년차 |
| 4~5 | 중간 |
| 6~8 | 고년차 |
| 9~10 | 최고년차/책임 가능 인력 |

실제 의미는 병원 상황에 맞게 정하면 됩니다. 중요한 것은 Grade 정책에서 고년차/저년차 기준을 어떻게 잡는지입니다.

### 6.5 근무 조정값

근무 조정값은 평균보다 몇 개 더 또는 덜 일하게 할지 조정하는 값입니다.

| 값 | 의미 |
|---:|---|
| 0 | 평균대로 |
| +1 | 평균보다 약 1개 더 |
| -1 | 평균보다 약 1개 덜 |

연차 `a`나 `fixed_Total`은 solver 내부에서 몰래 근무 조정값을 바꾸지 않습니다. 대신 편집 표의 `자동계산` 버튼으로 화면의 `근무조정값` 칸에 먼저 계산해 넣고, 확인 후 `저장`으로 반영할 수 있습니다.

자동계산 방식:

```text
fixed_Total이 있으면: 근무조정값 = fixed_Total - estimated average
fixed_Total이 없으면: 근무조정값 = -연차(a) 개수
차이가 1개 미만이면 0으로 처리
```

### 6.6 fixed_Total 표와 D/E/N 고정 개수

특정 의사의 전체 근무 수 또는 D/E/N 개수를 정확히 고정하고 싶을 때 사용합니다. `fixed_D / fixed_E / fixed_N / fixed_Total`은 `fixed D/E/N/Total / Duty 총합 확인` 요약 바로 아래의 편집 표에서 한 번에 수정합니다.

| 값 | 의미 |
|---:|---|
| 빈칸 | 자동 평준화 |
| 0 | 해당 항목을 0개로 고정 |
| 3 | 해당 항목을 정확히 3개로 고정 |

이전 config 파일에 남아 있는 `-1`은 자동 평준화로 계속 읽습니다.

예:

| fixed_D | fixed_E | fixed_N | fixed_Total | 의미 |
|---:|---:|---:|---:|---|
| -1 | -1 | -1 | -1 | 모두 자동 평준화 |
| -1 | -1 | -1 | 8 | 총근무는 정확히 8개, D/E/N 구성은 자동 |
| 3 | -1 | -1 | -1 | D는 정확히 3개, E/N/Total은 자동 |
| 3 | 3 | 4 | 10 | 총 10개, D/E/N도 3/3/4개로 고정 |

주의:

- fixed_Total은 hard rule입니다.
- fixed_D/E/N 합이 fixed_Total보다 크면 불가능합니다.
- fixed_D/E/N을 모두 고정한 경우 합이 fixed_Total과 같아야 합니다.
- 연차 `a`는 fixed_Total 값을 줄이지 않습니다. 필요하면 자동계산 버튼으로 근무조정값을 화면에 계산한 뒤 저장합니다.

### 6.7 개인별 근무 규칙

| 규칙 | 의미 |
|---|---|
| 하루 근무 횟수 | 하루에 D/E/N을 몇 개까지 허용할지 |
| N 뭉치 최대 길이 | 연속 Night 허용 길이 |
| N뭉치 후 완전 Off 의무일 | Night block 후 반드시 쉬어야 하는 날짜 수 |
| N뭉치 후 다음 N까지 총 간격 | 다음 Night까지 최소 간격 |
| Evening 후 Day 금지 | E 다음날 D 금지 |
| Evening 3연속 금지 | EEE 금지 |
| 4일내 Evening 3회 금지 | 4일 중 E 3회 이상 방지 |
| 최대 연속 근무일수 | 입력한 일수까지 연속 근무 허용 |
| 7일 구간 최대 근무수 | 어느 7일 구간에서도 허용할 최대 근무수 |
| Day 3연속 금지 | DDD 금지 |

`최대 연속 근무일수 = 4`이면 4일 연속 근무까지 허용되고, 5일 연속 근무는 금지됩니다.

## 7. 결과 탭

### 7.1 근무 통계

근무 통계는 Name 정렬이 아니라 입력 순서 그대로 표시됩니다.  
맨 왼쪽 `No`는 `근무 요청 / 날짜 설정` 탭 및 Excel 입력 순서 기준입니다.

현재 주요 컬럼은 다음과 같습니다.

| 컬럼 | 의미 |
|---|---|
| No | 입력 순서 index |
| Name | 의사 이름 |
| Grade | 개인 grade |
| Senior | 고년차 여부 또는 관련 표시 |
| Junior | 저년차 여부 또는 관련 표시 |
| 초저년차 | 초저년차 hard rule 대상 여부 |
| D | Day 근무 수 |
| E | Evening 근무 수 |
| N | Night 근무 수 |
| Total | D+E+N 총근무 수 |
| 연차 | 근무 요청의 `a` 개수 |
| Holiday | 토/일/공휴일 근무 수 |
| Fri_N | 금요일 Night 수 |
| 주간평균hr | 주당 평균 근무시간 추정치 |

`Tue_N`은 제거되었습니다.

### 7.2 편차 / penalty 정보

결과 화면에는 objective 구성요소가 표시됩니다.

```text
편차*가중치 합
최적 편차
허용 상한
balance penalty
k D/E
k1 휴일
k2 총근무
k3 N
k4 Grade편차
저년차 초과 penalty
```

값이 낮을수록 solver가 더 선호한 근무표입니다. 다만 실제 운영에서는 숫자만 보지 말고 특정 날짜의 인력 구성도 함께 확인해야 합니다.

### 7.3 근무 일정표 / 셀 선택 & 고정

근무 일정표는 Grade로 정렬하지 않고 입력 순서 그대로 표시됩니다.  
다만 이름 옆에 grade는 함께 표시됩니다.

```text
Kim [G8]
Lee [G3]
Park [G6]
```

날짜 열은 공간을 줄이기 위해 짧게 표시됩니다.

```text
1토, 2일, 3월, 4화, 5수, 6목, 7금
```

평일 공휴일은 `*`가 붙어 구분됩니다.

| 표시 | 의미 |
|---|---|
| D | Day |
| E | Evening |
| N | Night |
| A | 연차 |
| · | Off |
| 🔒D / 🔒E / 🔒N / 🔒· / 🔒A | 결과표에서 고정된 셀 |

### 7.4 셀 고정 / 고정 해제

결과표에서 셀을 선택하고 고정하면 다음 실행에서도 유지됩니다.

| 기능 | 설명 |
|---|---|
| 🔒 선택 셀 고정 | 현재 결과값을 fixed layer로 저장합니다. D/E/N은 해당 근무로, off는 `x`, 연차는 `a`로 고정됩니다. |
| 🔓 선택 셀 고정 해제 | fixed layer만 제거합니다. 기존 사용자가 입력했던 `d`, `en`, 빈칸 등은 다시 살아납니다. |

내부적으로 근무 요청은 두 층으로 관리됩니다.

```text
base_shift_requests  = 사용자가 근무 요청 탭 또는 Excel에서 직접 입력한 원래 값
fixed_shift_requests = 결과표에서 셀을 선택해 추가로 고정한 값
solver 입력값         = base_shift_requests 위에 fixed_shift_requests를 덮어쓴 값
```

예:

```text
원래 입력: en
결과표에서 D로 고정
→ solver에는 D가 들어감
고정 해제
→ 원래 en이 다시 적용됨
```

### 7.5 Daily duty 구성 요약

날짜별 D/E/N duty마다 실제 배정 인원, 필요 인원, 평균 grade, 근무자를 보여줍니다.

근무자 이름 옆에는 grade가 같이 표시됩니다.

```text
Kim [8], Lee [3], Park [6]
```

주요 컬럼:

| 컬럼 | 의미 |
|---|---|
| 날짜 | 날짜와 요일 |
| 유형 | 평일/토/일/공 |
| Duty | D/E/N |
| 인원/필요 | 실제 배정 인원 / 필요 인원 |
| 평균 Grade | 해당 duty 근무자의 평균 grade |
| 근무자 | 이름 [grade] 목록 |

토/일/공휴일 row는 음영 처리됩니다.

### 7.6 다중 솔루션 이동

`이전`, `다음`, `이동` 기능으로 이미 찾은 여러 후보를 비교할 수 있습니다.  
이미 계산된 solution을 넘겨보는 것이므로 solver를 새로 실행하지 않습니다.

### 7.7 결과 Excel 내보내기

결과 Excel은 화면 전환 때마다 자동 생성하지 않고, `Excel 준비` 버튼을 눌렀을 때 생성됩니다.  
준비가 끝나면 다운로드 버튼으로 저장합니다.

## 10. 추천 운영 방식

### 10.1 처음 세팅할 때

```text
1. 의사 이름을 모두 추가
2. 설정 Excel 저장
3. Excel에서 Doctors, Rules, DutyRequests, ShiftRequests 편집
4. 다시 불러오기
5. 앱에서 Grade 정책, fixed_Total, Duty 총합 요약 확인
6. 스케줄 생성
```

### 10.2 연차가 있는 달

```text
1. ShiftRequests에 연차 날짜를 a로 입력
2. 개인 규칙 / Grade 탭의 표에서 연차(a) 개수 확인
3. 필요하면 `자동계산` 버튼으로 근무조정값을 화면에 계산한 뒤 `저장`으로 반영
4. fixed_Total이 있는 사람은 Total이 줄지 않으므로 필요하면 fixed_Total을 직접 조정
5. 결과 통계에서 연차 컬럼 확인
```

### 10.3 fixed_D/E/N/Total을 많이 쓰는 달

```text
1. 개인별 Total 고정 입력
2. Duty 설정 탭 상단의 차이 확인
3. 차이 0이면 가장 안전
4. 차이 +값이면 남은 근무가 fixed_Total 없는 사람에게 배분됨
5. 차이 -값이면 불가능하므로 Duty를 늘리거나 fixed_Total을 줄임
```

## 11. 해가 안 나올 때 확인할 것

1. Duty 총합보다 fixed_total 합이 크지 않은지 확인합니다.
2. 모든 사람에게 fixed_Total을 넣었다면 합이 Duty 총합과 정확히 같은지 확인합니다.
3. fixed_D/E/N 합이 fixed_Total보다 크지 않은지 확인합니다.
4. 고년차 최소 인원이 duty 필요 인원보다 크지 않은지 확인합니다.
5. 고년차 기준에 해당하는 사람이 충분한지 확인합니다.
6. 초저년차 hard rule이 너무 엄격하지 않은지 확인합니다. 예: 초저년차가 많고 duty 필요 인원이 큰데 `초저년차 최대 허용 = 1`로 설정하면 해가 없을 수 있습니다.
7. 대문자 `D/E/N` 강제 요청이 너무 많지 않은지 확인합니다.
8. `x`, `a`, `den`으로 전체 불가인 날이 너무 많지 않은지 확인합니다.
9. N 후 휴식, 최대 연속 근무, 7일 구간 최대 근무수가 너무 엄격하지 않은지 확인합니다.
10. solver 탐색 시간을 늘려봅니다.

## 12. 결과가 마음에 안 들 때 조정법

| 문제 | 조정할 항목 |
|---|---|
| 총근무 수가 불공평 | 총 근무 편차 가중치 증가 |
| Night가 불공평 | N 편차 가중치 증가 |
| 휴일이 불공평 | 휴일 편차 가중치 증가 |
| D/E가 불공평 | D/E 편차 가중치 증가 |
| duty별 grade 평균이 너무 낮음 | Grade 편차 가중치 증가, 고년차 최소 인원 조정 |
| 저년차가 너무 몰림 | 저년차 penalty 증가 또는 초저년차 hard rule 사용 |
| 저년차 penalty가 너무 지배적 | 저년차 penalty 감소 |
| 초저년차가 같은 duty에 같이 들어가면 안 됨 | 초저년차 기준 grade와 `초저년차 최대 허용` 설정 |
| 특정 사람이 평균보다 많이/적게 해야 함 | 근무 조정값 또는 fixed_Total 조정 |
| 연차자가 너무 많이 배정되는 느낌 | 연차는 `a`, 전체 불가는 `x`로 정확히 입력했는지 확인 |

## 13. 주의사항

- 이 프로그램은 최적화 도구이며 최종 승인자가 아닙니다. 생성된 근무표는 반드시 실제 운영자가 검토해야 합니다.
- Hard rule이 너무 많으면 해가 없을 수 있습니다.
- Soft rule은 완벽히 지켜지는 조건이 아니라 penalty를 통해 유도되는 조건입니다.
- 다중 솔루션의 1단계 최적 편차는 solver가 주어진 탐색 시간 안에서 찾은 최선값입니다. 탐색 시간이 너무 짧으면 더 좋은 해가 있을 수 있습니다.
- Excel에서 의사 이름을 바꾸면 ShiftRequests / FixedShiftRequests 매칭이 달라질 수 있습니다.
- 앱 화면에서 설정 Excel을 불러온 뒤에는 Duty 설정 탭과 개인 규칙 / Grade 탭에 값이 제대로 반영되었는지 확인하세요.
- grade는 1~10 범위로 사용하는 것을 권장합니다.

## 15. 진단모드: 해가 없을 때 병목 후보 확인

스케줄 생성 결과가 없을 때, 앱은 자동으로 진단모드에 들어가지 않습니다. 결과 탭에 **진단모드 실행** 버튼이 나타나며, 사용자가 눌렀을 때만 진단을 실행합니다.

진단모드는 최적해를 만드는 기능이 아니라, hard rule 때문에 해가 막히는 병목 후보를 찾는 기능입니다.

### 진단모드 실행 흐름

```text
1. 사이드바에서 스케줄 생성 클릭
2. 조건을 만족하는 해가 없으면 결과 탭에 실패 메시지 표시
3. 결과 탭의 🧪 진단모드 실행 버튼 클릭
4. 1차 / 2차 병목 후보 표 확인
```

### 1차 진단: 날짜 / Duty별 후보 부족

각 날짜와 D/E/N duty마다 다음을 확인합니다.

```text
- 필요 인원
- 실제 가능한 후보 수
- 고년차 후보 수
- 초저년차 후보 수
- 초저년차 최대 허용 rule 적용 후 가능한 인원
- hard fixed 인원 수
- 제외된 주요 원인
```

예시는 다음과 같습니다.

```text
날짜      Duty   필요   가능후보   고년차후보   주요 이슈
08/12    N      3      2          1           가능 후보 2명 < 필요 3명
08/15    E      4      5          0           고년차 후보 0명 < 최소 1명
08/18    D      3      5          2           초저년차 제한 적용 후 가능 2명 < 필요 3명
```

제외 주요 원인에는 다음과 같은 내용이 표시될 수 있습니다.

```text
연차(a) 2명
x/전체불가 1명
N 불가 3명
다른 duty 고정(D) 1명
```

### 1차 진단: 개인 fixed_Total / fixed_D/E/N 가능성

개인별 고정 근무 수가 실제 가능 범위를 넘는지도 확인합니다.

예:

```text
이름     항목          고정값   단순 가능 최대   부족
Kim      fixed_Total   18       15              3
Lee      fixed_N       7        5               2
```

이 경우에는 해당 사람의 fixed count를 줄이거나, 불가 요청/연차/고정 셀을 완화해야 합니다.

### 2차 진단: 구간별 병목 후보

하루 단위로는 가능해 보여도, 특정 구간 전체에서 근무 수가 부족할 수 있습니다. 진단모드는 다음 구간을 확인합니다.

```text
- 3일 구간
- 5일 구간
- 7일 구간
- 전체 기간
```

각 구간에서 다음을 비교합니다.

```text
필요 근무수 vs 단순 가능 최대
```

예:

```text
구간          항목       필요   단순 가능 최대   부족
08/10~08/16   전체근무   154    149             5
08/18~08/22   N          15     12              3
```

이 표는 “이 구간에서 구조적으로 몇 개가 부족해 보이는지”를 보여줍니다.

### 진단모드의 한계

진단모드는 1차 / 2차 병목 후보만 찾습니다. 즉, 다음과 같은 복잡한 전체 조합 충돌은 완벽하게 증명하지 못할 수 있습니다.

```text
- N 뭉치 후 rest / gap
- Evening 후 Day 금지
- 최대 연속 근무일수
- 여러 사람의 fixed_Total이 동시에 얽힌 경우
- 다중 shift 허용 rule이 복잡하게 작동하는 경우
```

따라서 진단 결과는 다음처럼 이해하는 것이 좋습니다.

```text
정답 원인 하나를 확정하는 표가 아니라,
해가 막히는 주요 병목 후보를 보여주는 표
```

진단 결과에 아무 병목이 안 보이는데도 해가 없다면, 그때는 3차 진단인 relax solver 방식이 필요할 수 있습니다. 현재 버전은 요청대로 2차 진단까지만 포함합니다.

## 최근 수정: fixed count 편집표 즉시 반영

개인 규칙 / Grade 탭의 fixed count 편집표에서 `근무조정값`, `fixed_D`, `fixed_E`, `fixed_N`, `fixed_Total`을 수정한 뒤 저장하면, 최신 값이 스케줄 생성과 진단모드 계산에 반영되도록 보강했습니다.

수정 내용:

- fixed count 편집표의 변경값을 solver 실행 직전에 다시 동기화합니다.
- Duty 총합 / fixed_total 요약 계산 전에 fixed count 편집표의 변경값을 다시 읽습니다.
- 진단모드 실행 전에도 같은 동기화를 수행합니다.
- Streamlit `data_editor`의 edit-delta 형식과 DataFrame 형식을 모두 안전하게 처리합니다.

이제 fixed count 표에서 값을 바꾼 뒤 바로 `스케줄 생성`을 눌러도 이전 값으로 계산되는 문제가 줄어듭니다.


### 최신 변경: 미지정 인원 1명당 자동 배분 예상치 표시

Duty 총합과 fixed_total 합의 차이가 양수이고 fixed_Total 미지정 인원이 있으면, 요약 문구에 `(약 N.N개/명)`을 함께 표시합니다. 예를 들어 `534개 / 29명 = 약 18.4개/명`처럼 보여주므로, 미지정 인원에게 fixed_Total을 대략 몇 개로 넣으면 좋을지 판단하기 쉽습니다.


## 최근 수정: 근무 조정값을 fixed count 편집표에 통합

개인 규칙 / Grade 탭의 `fixed D/E/N/Total / Duty 총합 확인` 영역에서, `초저년차` 컬럼과 `fixed_D` 컬럼 사이에 `근무조정값` 컬럼을 추가했습니다.

```text
No | Name | Grade | Senior | Junior | 초저년차 | 연차(a) | 근무조정값 | fixed_D | fixed_E | fixed_N | fixed_Total
```

이제 연차(a), 근무 조정값과 fixed count를 한 표에서 같이 확인/수정한 뒤 `저장` 버튼으로 한 번에 반영합니다. `자동계산` 버튼을 누르면 권장값이 화면의 근무조정값 칸에만 입력되고, 실제 반영은 `저장`을 눌렀을 때 이루어집니다. 아래 개인별 rule 표에서는 중복을 피하기 위해 근무 조정값 입력줄을 제거했습니다.


## 최근 수정: 연차/fixed_Total 보정을 명시적 shift_adj 버튼으로 전환

이 버전부터 `a` 연차나 `fixed_Total`이 solver 내부에서 몰래 `shift_adj`처럼 계산되지 않습니다.

```text
solver 내부:
a = 해당 날짜 근무 불가 + 연차 통계 표시
fixed_Total = 총근무 hard 고정
shift_adj = 표에 실제 입력된 근무조정값만 사용
```

개인 규칙 / Grade 탭의 편집 표에는 `연차(a)`와 `근무조정값`이 함께 표시됩니다.

```text
fixed_Total이 있으면 자동계산값 = fixed_Total - estimated average
fixed_Total이 없으면 자동계산값 = -연차(a) 개수
차이가 1개 미만이면 0
```

`자동계산` 버튼을 누르면 이 자동계산값이 화면의 `근무조정값` 칸에만 입력됩니다. 실제 설정에는 아직 저장되지 않으므로, 확인 후 `저장` 버튼을 눌러 반영합니다. `저장` 버튼은 현재 화면 표의 근무조정값과 fixed count 값을 그대로 저장합니다.


## 최근 수정: 연차(a) 표시 강화

개인 규칙 / Grade 탭의 근무조정값 & fixed count 표 안내 문구에 `연차(a)` 개수를 명확히 포함했습니다.

```text
No | Name | Grade | Senior | Junior | 초저년차 | 연차(a) | 근무조정값 | fixed_D | fixed_E | fixed_N | fixed_Total
```

또한 Duty 총합 / fixed_total 확인 요약 박스에도 `연차(a) 총합`을 함께 표시합니다.


## 최근 수정: 근무 통계 Total+연차 추가

결과 탭의 근무 통계에서 `Total` 옆에 `연차`와 `Total+연차`를 함께 표시합니다.
