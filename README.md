# SHIFT SCHEDULER 사용 설명서

의료진 D/E/N 근무표를 자동으로 생성하는 스케줄러입니다.  
날짜별 필요 인원, 개인별 불가/희망 근무, 야간 후 휴식, 연속근무 제한, grade 기반 고년차 배정, 저년차 분산, 특정 shift 개수 고정 등을 반영하여 OR-Tools CP-SAT solver로 근무표를 만듭니다.

---

## 1. 기본 개념

이 프로그램은 아래 항목들을 나누어 생각합니다.

| 항목 | 의미 | 예시 |
|---|---|---|
| Duty 설정 | 날짜별로 D/E/N에 필요한 인원 수 | 7/1 D 3명, E 2명, N 1명 |
| 근무요청/날짜설정 | 개인별 특정 날짜에 못하는/원하는 근무 | 소문자는 못하는 날, 대문자는 강제 배정, x는 off: `d`, `x`, `N` |
| 개인 규칙 | 각 의사별 근무 패턴 제한 | N 후 off, 연속 N 개수, 최대 연속 근무일수 등 |
| Grade 정책 | 전체 스케줄에 적용되는 고년차/저년차 rule | 고년차 기준과 duty별 의무 고년차 배정 명수, 저년차 기준과 duty별 저년차 배치 제한 |
| Objective/penalty | 가능한 한 공평하게 맞추는 기준을 계산하는 변수 | D/E/N 개수, 휴일, 총근무 편차, duty별 저년차 초과 |

### Hard rule과 Soft rule

| 종류 | 설명 | 예시 |
|---|---|---|
| Hard rule | 반드시 지켜야 하는 조건. 불가능하면 해가 나오지 않음 | 필요한 D/E/N 인원, 불가 근무, 고년차 최소 인원 |
| Soft rule | 가능하면 맞추는 조건. 어기면 penalty가 붙음 | 저년차가 한 duty에 너무 몰리지 않게, 근무 편차 최소화 |

## 2. 전체 작업 순서

권장 순서는 아래와 같습니다.

```text
1. 시작 날짜와 총 일수 설정
   **엑셀에 지정된 날짜가 아닌 sidebar의 시작날짜를 기준으로 달력을 계산함**
2. 의료진 이름 추가 또는 Excel 설정 파일이 있으면 불러오기
3. 날짜 유형과 개인 근무 요청 입력
4. 날짜별 D/E/N 필요 인원 입력
5. Grade 정책과 개인별 규칙 입력
6. 스케줄 생성
7. 결과 확인
8. 필요하면 다중 솔루션 비교
9. 결과 Excel 저장
```

처음 사용하는 경우에는 먼저 의사 이름만 추가한 뒤, `설정 Excel 저장`을 눌러 기본 템플릿을 만들고 Excel에서 대량 편집하는 방식이 가장 편합니다.

## 3. 사이드바 사용법

### 3.1 기본 설정

| 항목 | 설명 |
|---|---|
| 시작 날짜 | 스케줄의 첫 날짜 |
| 총 일수 | 생성할 기간. 보통 한 달 단위 |

시작 날짜를 바꾸면 토/일 날짜 유형이 자동 재계산됩니다.

### 3.2 의사 관리

왼쪽 사이드바에서 의료진 이름을 추가합니다.

- 이름은 중복될 수 없습니다.
- 의사를 삭제하면 해당 의사의 rule과 요청도 index 기준으로 바뀔 수 있으므로, 실제 운영 중에는 삭제보다 Excel에서 정리 후 다시 불러오는 방식을 권장합니다.
- Grade와 근무 조정값은 `개인 규칙 / Grade` 탭에서 설정합니다.

### 3.3 솔버 설정

| 모드 | 설명 |
|---|---|
| 최적해 1개 | 가장 좋은 해 하나를 찾습니다. 날짜별 duty, 원하는 off, 해당 일자 고년차 최소 배치 수 등의 Hard rule을 만족하면서 soft rule penalty가 최소화되는 해입니다. 일반적으로 가장 빠르고 안정적입니다. |
| 다중 솔루션 탐색 | 먼저 최적 penalty를 찾은 뒤, 그 최적값과 같거나 사용자가 허용한 범위 안의 여러 근무표를 찾아 비교합니다. |

다중 솔루션 탐색의 변수는 아래와 같습니다.

| 항목 | 설명 |
|---|---|
| 최대 탐색 시간 | solver가 해를 찾는 데 사용할 최대 시간 |
| 최대 솔루션 수 | 다중 솔루션 모드에서 저장할 후보 수 |
| 최소 편차에 추가 허용 편차 | solver가 먼저 찾은 최적 penalty보다 얼마나 더 나쁜 해까지 후보에 포함할지 결정 |

다중 솔루션 탐색은 2단계로 작동합니다.

```text
1단계: solver가 먼저 soft rule penalty의 최적값(best_adv)을 찾음
2단계: adv <= best_adv + 추가 허용 편차 범위 안에서 여러 solution을 탐색
```

예:

```text
최적 편차(best_adv) = 21
추가 허용 편차 = 0  → adv <= 21인 최적해만 여러 개 탐색
추가 허용 편차 = 2  → adv <= 23까지 허용하여 조금 더 다양한 후보 탐색
```

다중 솔루션에서 `이전`, `다음`을 눌러 다중 검색한 후보들을 확인합니다. 이미 찾은 solution을 넘겨보는 것이므로 solver를 새로 실행하지 않습니다.

## 4. 탭별 사용법

## 4.1 근무 요청 / 날짜 설정

이 탭에서는 날짜 유형과 개인별 근무 요청을 입력합니다.

### 날짜 유형

| 값 | 의미 |
|---|---|
| 평일 | 일반 평일 |
| 토 | 토요일 |
| 일 | 일요일 |
| 공 | 공휴일 |

`토`, `일`, `공`은 휴일 근무 통계에 포함됩니다.

### 개인 근무 요청 입력값

| 입력 | 의미 |
|---|---|
| 빈칸 | 특별 요청 없음 |
| `d` | Day 불가 |
| `e` | Evening 불가 |
| `n` | Night 불가 |
| `de` | Day, Evening 불가 |
| `dn` | Day, Night 불가 |
| `en` | Evening, Night 불가 |
| `den` | D/E/N 모두 불가 |
| `x` | 해당 날짜 전체 불가 |
| `D` | Day 희망 또는 강제 |
| `E` | Evening 희망 또는 강제 |
| `N` | Night 희망 또는 강제 |

주의: 대문자 `D/E/N`은 solver에서 해당 근무를 반드시 배정해야 하는 요청으로 처리됩니다. 너무 많이 넣으면 해가 안 나올 수 있습니다.

---

## 4.2 Duty 설정

날짜별로 필요한 D/E/N 인원을 입력합니다.

예:

| 날짜 | D | E | N |
|---|---:|---:|---:|
| 7/1 | 3 | 2 | 1 |
| 7/2 | 3 | 2 | 1 |
| 7/3 | 2 | 1 | 1 |

필요 인원이 0인 duty에는 아무도 배정되지 않습니다.

## 4.3 개인 규칙 / Grade

이 탭은 가장 중요합니다.  
전체 grade 정책, 편차 가중치, 개인별 grade, shift count 고정, 개인 rule을 설정합니다.

---

### A. Grade 정책 설정

| 항목 | 의미 | 추천 시작값 |
|---|---|---:|
| 고년차 기준 grade ≥ | 이 grade 이상이면 고년차 | 2 |
| Duty별 고년차 최소 | 각 D/E/N duty에 필요한 고년차 최소 인원 | 1 |
| 저년차 기준 grade ≤ | 이 grade 이하이면 저년차 | 1 |
| Duty별 저년차 권장 최대 | 한 duty에 들어가도 좋은 저년차 수 | 1 |
| 저년차 초과 penalty | 저년차가 권장 최대를 넘을 때 penalty | 1 |

예를 들어 `고년차 기준 grade ≥ 2`, `Duty별 고년차 최소 1`이면, 모든 D/E/N duty에 grade 2 이상이 최소 1명 들어갑니다. 이 조건은 hard rule입니다.

저년차 제한은 soft rule입니다. 예를 들어 저년차 권장 최대가 1인데 한 duty에 저년차 2명이 들어가면 penalty가 붙습니다. 하지만 다른 조건 때문에 필요하면 허용됩니다.

---

### B. 편차 가중치 설정

근무표가 여러 개 가능할 때 어떤 균형을 더 중요하게 볼지 정합니다.

| 항목 | 의미 | 기본값 |
|---|---|---:|
| D/E 편차 가중치 | D와 E 개수 불균형을 줄이는 정도 | 1 |
| 휴일 편차 가중치 | 토/일/공휴일 근무 불균형을 줄이는 정도 | 3 |
| 총 근무 편차 가중치 | 전체 근무 개수 불균형을 줄이는 정도 | 5 |
| N 편차 가중치 | Night 개수 불균형을 줄이는 정도 | 5 |

가중치를 높이면 그 항목의 균형을 더 강하게 맞추려고 합니다.  
인원이 많아지고 저년차 penalty가 너무 크게 보이면 `저년차 초과 penalty`를 낮추거나, 총근무/N/휴일 가중치를 상대적으로 올려 조정합니다.

---

### C. 개인별 Grade

각 의사의 grade를 입력합니다.

| Grade | 일반적 의미 예시 |
|---:|---|
| 1 | 저년차 |
| 2 | 독립 근무 가능 |
| 3 | 중간 이상 |
| 4 | 고년차 |
| 5 | 최고년차 |

Grade의 실제 의미는 병원 상황에 맞게 정하면 됩니다. 중요한 것은 `Grade 정책 설정`에서 고년차/저년차 기준을 어떻게 잡느냐입니다.

---

### D. 근무 조정값

근무 조정값은 평균보다 몇 개 더 또는 덜 일하게 할지 정하는 값입니다.

| 값 | 의미 |
|---:|---|
| 0 | 평균대로 |
| +1 | 평균보다 약 1개 더 |
| -1 | 평균보다 약 1개 덜 |

근무 조정값은 총근무뿐 아니라 D/E/N 및 휴일 목표에도 비율대로 영향을 줍니다.

예:

```text
평균 총근무가 8개인데 어떤 사람이 10개 정도 해야 한다면 shift_adj = +2
평균 총근무가 8개인데 어떤 사람이 6개 정도 해야 한다면 shift_adj = -2
```

### E. Total 및 D/E/N 고정 개수

특정 의사의 전체 근무 수 또는 D/E/N 개수를 정확히 고정하고 싶을 때 사용합니다.

| 값 | 의미 |
|---:|---|
| -1 | 자동 평준화 |
| 0 | 해당 항목을 0개로 고정 |
| 3 | 해당 항목을 정확히 3개로 고정 |

예:

| Total 고정 | D 고정 | E 고정 | N 고정 | 의미 |
|---:|---:|---:|---:|---|
| -1 | -1 | -1 | -1 | 모두 자동 평준화 |
| 8 | -1 | -1 | -1 | 총근무는 정확히 8개, D/E/N 구성은 자동 |
| -1 | 3 | -1 | -1 | D는 정확히 3개, E/N/Total은 자동 |
| 10 | 3 | 3 | 4 | 총 10개이면서 D/E/N을 각각 3/3/4개로 고정 |

D/E/N을 모두 고정한 사람은 총근무 개수가 이미 정해지므로 D/E/N/Total 편차에서는 대부분 제외되고, 휴일 균형은 여전히 고려됩니다.  
이 사람이 평균보다 많이 또는 적게 일하도록 고정된 경우에는 근무 조정값을 같이 조정하는 것이 좋습니다.

#### fixed_total 사용법

`Total 고정`은 한 사람의 전체 근무 수를 hard rule로 고정합니다. `-1`이면 자동 평준화입니다.

예를 들어 `Total 고정 = 8`이면 그 의사는 D/E/N 합계가 반드시 8개가 됩니다. D/E/N 각각은 별도로 고정하지 않았다면 solver가 자동으로 배분합니다.

Duty 설정 탭 상단에는 다음 값이 항상 표시됩니다. Excel 설정 파일을 불러온 직후에도 이 요약을 먼저 확인하면 전체 근무 수가 맞는지 바로 알 수 있습니다.

```text
Duty 총합 = 날짜별 D/E/N 필요 인원 전체 합
fixed_total 합 = Total 고정을 입력한 사람들의 합
남은 근무수 = Duty 총합 - fixed_total 합
```

- fixed_total이 없는 사람이 있으면, 남은 근무수는 그 사람들에게 자동 배분됩니다.
- 모든 사람의 fixed_total을 입력한 경우에는 fixed_total 합이 Duty 총합과 정확히 같아야 합니다.
- fixed_total 합이 Duty 총합보다 크면 해가 나오지 않으므로 Duty 설정에서 근무 수를 추가하거나 fixed_total을 줄여야 합니다.
- fixed_D/E/N 합이 fixed_total보다 크면 불가능합니다.
- 설정 Excel을 다시 불러오면 Duty 입력칸과 Total/D/E/N 고정 개수 입력칸이 새 설정값으로 즉시 갱신됩니다.

### F. 개인별 근무 규칙

| 규칙 | 의미 |
|---|---|
| 하루 근무 횟수 | 하루에 D/E/N을 몇 개까지 허용할지 |
| N 뭉치 최대 길이 | 연속 Night 허용 길이 |
| N뭉치 후 완전 Off 의무일 | Night block 후 반드시 쉬어야 하는 날짜 수 |
| N뭉치 후 다음 N까지 총 간격 | 다음 Night까지 최소 간격 |
| Evening 후 Day 금지 | E 다음날 D 금지 |
| Evening 3연속 금지 | EEE 금지 |
| 4일내 Evening 3회 금지 | 4일 중 E 3회 이상 방지 |
| 최대 연속 근무일수 | 입력한 일수까지 연속 근무 허용 |
| 7일 구간 최대 근무수 | 어느 7일 구간에서도 허용할 최대 근무수 |
| Day 3연속 금지 | DDD 금지 |

#### 최대 연속 근무일수

`최대 연속 근무일수 = 4`이면 4일 연속 근무까지는 허용되고, 5일 연속 근무는 금지됩니다.

#### 7일 구간 최대 근무수

`7일 구간 최대 근무수 = 6`이면 어느 7일 구간에서도 최대 6일 또는 6개 shift까지 허용됩니다.  
현재 하루 1근무 원칙을 쓰면 보통 7일 중 6일 근무와 비슷하게 해석됩니다.

## 5. 결과 탭 사용법

스케줄 생성 후 결과 탭에서 다음을 확인합니다.

### 근무 통계

각 의사의 D/E/N/Total/Holiday 등을 보여줍니다.

| 컬럼 | 의미 |
|---|---|
| D | Day 근무 수 |
| E | Evening 근무 수 |
| N | Night 근무 수 |
| Total | 총 근무 수 |
| Holiday | 토/일/공휴일 근무 수 |
| Tue_N | 화요일 Night 수 |
| Fri_N | 금요일 Night 수 |
| 주간평균hr | 주당 평균 근무시간 추정치 |

### 편차와 penalty

결과 화면에는 objective 구성요소가 표시됩니다.

```text
편차*가중치 합
최적 편차
허용 상한
D/E 편차
휴일 편차
N 편차
총근무 편차
저년차 초과 penalty
```

값이 낮을수록 solver가 더 선호한 근무표입니다. 다만 실제 운영에서는 숫자만 보지 말고 특정 날짜의 인력 구성이 현실적인지도 함께 확인하세요.

### 근무 일정표

근무 일정표는 grade 높은 순서로 표시됩니다.  
Grade가 바뀌는 지점에는 구분 줄이 들어갑니다.

| 표시 | 의미 |
|---|---|
| D | Day |
| E | Evening |
| N | Night |
| · | Off |

### 다중 솔루션 이동

`이전`, `다음` 버튼으로 여러 solution을 비교합니다.  
이미 찾은 solution을 넘겨보는 것이므로 solver를 새로 실행하지 않습니다.

### Excel 내보내기

결과 Excel은 화면 전환 때마다 만들지 않고, `Excel 준비`를 눌렀을 때 생성됩니다.  
준비가 끝나면 다운로드 버튼으로 저장합니다.

## 7. 추천 운영 방식

### 처음 세팅할 때

```text
1. 의사 이름을 모두 추가
2. 설정 Excel 저장
3. Excel에서 Doctors, Rules, DutyRequests, ShiftRequests를 편집
4. 다시 불러오기
5. 앱에서 Grade 정책과 rule을 확인
6. 스케줄 생성
```

### 해가 안 나올 때 확인할 것

1. 고년차 최소 인원이 duty 필요 인원보다 크지 않은지 확인합니다.  
2. 대문자 `D/E/N` 강제 요청이 너무 많지 않은지 확인합니다.  
3. `x` 또는 `den`으로 전체 불가를 너무 많이 넣지 않았는지 확인합니다.  
4. N 후 휴식, 최대 연속 근무, 7일 구간 최대 근무수가 너무 엄격하지 않은지 확인합니다.  
5. fixed_D/E/N 합이 전체 duty 수와 맞는지 확인합니다.  
6. solver 탐색 시간을 늘려봅니다.

### 결과가 너무 불공평할 때

| 문제 | 조정할 항목 |
|---|---|
| 총근무 수가 불공평 | 총 근무 편차 가중치 증가 |
| Night가 불공평 | N 편차 가중치 증가 |
| 휴일이 불공평 | 휴일 편차 가중치 증가 |
| 저년차가 너무 몰림 | 저년차 penalty 증가 |
| 저년차 penalty가 너무 지배적 | 저년차 penalty 감소 |
| 특정 사람이 평균보다 많이/적게 해야 함 | 근무 조정값 조정 |

## 8. 주의사항

- 이 프로그램은 최적화 도구이지 최종 승인자가 아닙니다. 생성된 근무표는 반드시 실제 운영자가 검토해야 합니다.
- Hard rule이 너무 많으면 해가 없을 수 있습니다.
- Soft rule은 완벽히 지켜지는 조건이 아니라 penalty를 통해 유도되는 조건입니다.
- 다중 솔루션은 여러 후보를 비교하기 위한 기능입니다. 첫 번째 solution이 항상 현장에서 가장 좋은 표라는 뜻은 아닙니다.
- Excel에서 날짜 열이나 의사 이름을 크게 바꾸면 불러오기 매칭이 달라질 수 있으므로 주의하세요.
- 다중 솔루션의 1단계 최적 편차는 solver가 주어진 탐색 시간 안에서 찾은 최선값입니다. 탐색 시간이 너무 짧으면 더 좋은 해가 있을 수 있습니다.

## 결과표에서 셀 고정 / 고정 해제

스케줄을 생성한 뒤 결과 탭에서 원하는 셀을 선택해 현재 결과를 다음 실행에도 유지할 수 있습니다.

| 기능 | 설명 |
|---|---|
| 🔒 선택 셀 고정 | 선택한 결과 셀의 현재 값을 wanted schedule의 hard layer로 고정합니다. D/E/N은 해당 근무로 고정되고, off 셀은 `x`로 고정됩니다. |
| 🔓 선택 셀 고정 해제 | 결과표에서 추가로 고정한 hard layer만 제거합니다. 기존 사용자가 입력했던 `d`, `en`, 빈칸 등의 원래 값은 다시 살아납니다. |

내부적으로 wanted schedule은 두 층으로 관리됩니다.

```text
base_shift_requests  = 사용자가 근무 요청 탭이나 Excel에서 직접 입력한 원래 값
fixed_shift_requests = 결과표에서 셀을 선택해 나중에 고정한 값
solver 입력값         = base_shift_requests 위에 fixed_shift_requests를 덮어쓴 값
```

예를 들어 사용자가 원래 `en`을 입력한 셀이 있었는데, 결과표에서 그 셀을 `D`로 고정하면 solver에는 `D`가 들어갑니다. 나중에 고정 해제를 하면 `D` layer만 제거되고 원래 `en`이 다시 적용됩니다.

이미 hard-coded 된 셀은 결과표와 선택용 표에서 `🔒` 표시가 붙습니다. 별도 배경색은 사용하지 않습니다.

근무 요청 / 날짜 설정 탭에서도 결과표에서 고정된 셀은 `🔒 fixed` 표시와 함께 연한 경고색으로 표시되고, selectbox가 비활성화됩니다. 이 상태에서는 원래 request 값을 직접 바꿀 수 없으므로, 먼저 결과 탭에서 `🔓 선택 셀 고정 해제`를 한 뒤 request를 수정하세요. 이렇게 하면 `D`로 고정된 셀에 사용자가 실수로 `de` 같은 충돌 요청을 입력해서 solver가 infeasible 되는 상황을 줄일 수 있습니다.

## 결과 탭 업데이트

결과 탭에서는 `근무 일정표`와 `셀 선택 & 고정 후 재실행` 표를 따로 보여주지 않고, 하나의 통합 근무표에서 바로 확인과 선택을 함께 합니다.

- 날짜 컬럼에는 `MM/DD(요일)` 형식으로 표시됩니다.
- 토/일은 `휴`, 직접 공휴일로 지정한 날짜는 `공`으로 표시됩니다.
- `🔒`가 붙은 셀은 base request 또는 fixed layer로 하드코딩된 셀입니다.
- 셀을 선택한 뒤 `🔒 선택 셀 고정`을 누르면 현재 결과값이 fixed layer에 저장됩니다.
- `🔓 선택 셀 고정 해제`를 누르면 fixed layer만 삭제되고, 원래 사용자가 입력했던 `d`, `en`, 빈칸 등이 다시 적용됩니다.

결과 탭 아래의 `Daily duty 구성 요약`에서는 날짜별 D/E/N에 대해 다음 정보를 확인할 수 있습니다.

| 항목 | 의미 |
|---|---|
| 인원/필요 | 실제 배정 인원 / 설정된 필요 인원 |
| 평균 Grade | 해당 duty에 배정된 사람들의 평균 grade |
| 근무자 | 해당 duty에 배정된 사람 이름 |

## 결과표 휴일 표시 업데이트

- 토요일/일요일은 요일 자체로 구분되므로 날짜 머리글에 별도 `휴` 표시를 붙이지 않습니다.
- 평일을 날짜 유형에서 `공`으로 지정한 경우에만 `MM/DD(월·휴)`처럼 표시합니다.
- 결과표와 Daily duty 구성 요약에서 평일 공휴일 컬럼/행은 은은한 음영으로 표시됩니다.

결과 탭의 `근무 일정표 / 셀 선택 & 고정` 표는 `근무 요청 / 날짜 설정` 탭에서 보이는 의사 입력 순서 그대로 표시됩니다. Grade는 이름 옆에 `[G2]`처럼 표시되지만, Grade 값으로 재정렬하지 않습니다. 결과 Excel의 Schedule sheet도 같은 입력 순서를 따르며, Grade는 별도 컬럼으로 함께 저장됩니다.

## 이번 버전 업데이트: fixed_total 요약 가독성 및 차이 안내

- Duty 설정 탭의 `Duty 총합 / fixed_total 합 / 남은 근무수` 요약 박스 글자를 흰색으로 바꿔 어두운 배경에서도 잘 보이도록 수정했습니다.
- `차이` 값을 `+2`, `-2`처럼 부호가 보이도록 표시합니다.
- 차이에 따라 아래와 같이 직접 조정 방향을 안내합니다.
  - `차이 +2`: Duty 총합이 fixed_total 합보다 2개 많음 → Duty를 2개 줄이거나 fixed_total을 2개 늘림
  - `차이 -2`: fixed_total 합이 Duty 총합보다 2개 많음 → Duty를 2개 늘리거나 fixed_total을 2개 줄임

## 업데이트: 연차 `a` 입력

근무 요청 / 날짜 설정 탭에서 `a`를 입력할 수 있습니다.

- `a` = 연차
- solver 로직에서는 `x`처럼 해당 날짜의 D/E/N 모두 불가로 처리됩니다.
- 단, 평준화에서는 `a` 1개당 근무 조정값 `-1`과 같은 효과를 줍니다.
  - 즉 연차가 1개 있으면 실제 배정 근무가 다른 사람보다 약 1개 적게 배정되도록 유도합니다.
- `fixed_total` hard constraint에는 영향을 주지 않습니다.
  - 예: fixed_Total = 10이면 연차가 있어도 실제 D/E/N 근무 총합은 10개로 맞춥니다.
- 결과 근무표와 Excel Schedule sheet에서는 연차가 `A`로 표시됩니다.
- 근무 통계에는 `Total` 옆에 `연차` 컬럼이 표시됩니다.
- `Tue_N` 통계 컬럼은 제거했습니다. `Fri_N`은 유지됩니다.


## 추가 수정: 근무 통계 연차 표시 보강

- 근무 통계 표에 `연차` 컬럼이 항상 표시됩니다.
- `연차`는 현재 근무 요청의 `a` 개수를 기준으로 계산되며, `Total` 바로 오른쪽에 표시됩니다.
- 이전 solve 결과나 legacy summary에 `연차` 컬럼이 없더라도 app에서 다시 계산해 표시합니다.
- Excel export의 `Summary` sheet에도 같은 `연차` 컬럼이 포함됩니다.
